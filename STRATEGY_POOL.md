## ⭐ OVERLOAD 超载机制（沙盒实测，关键）
- `me.overload()` 无参数,给**下一发**装填**两颗子弹**;施放消耗当帧动作(本帧装填/下帧开火);装填保持≤10帧;**CD 32**。
- **两颗子弹相隔一条车道(实测)**:朝向**左/右**→覆盖本行 y 和 **y+1 行**;朝向**上/下**→覆盖本列 x 和 **x+1 列**。
- => **超载=两车道宽射击**。能打在我正线、或**+垂直方向偏1车道**的敌人(「跨行/跨列」击杀,普通坦克打不到的角度),也能补到侧移躲进那条车道的敌人。
- 用法:正线敌开火前先装填(封住相邻闪避车道);跨行敌(+垂直相邻车道,此时敌无法直射我=安全)转向装填发两车道射击。
- **威力实测:opv1 首批 12-0,RS 848→1132(gold1→白金1),碾压白金高迭代坦克。远超 poison 版天花板(~gold,RS~880)。**

## 已确认的 poison 机制（实测）
- `me.poison()` 无参数，给敌挂 **4 帧** debuff，CD≈20，无需对齐/视线/距离（敌 enemy.tank 非 null 即可）。施放消耗当帧动作。
- **是"减速/半卡帧"而非完全定身**：复盘 RoutineEgg 局，我 f19 下毒，敌 f20 仍开火 → 中毒的敌人**仍可能开火/移动，只是节奏变慢**。
  → poison = **命中放大器**（敌躲子弹的机会变少），**不是保底击杀/保底防守**。别拿它当贴脸救命按钮。

## 已知败因 / 待防（复盘归纳）
- **L-A 被动零射击致死**：3败全是 shots 0/0 被 crashed。→ 需更主动开火 + 别把帧浪费在无兑现的 poison 上。
- **L-B poison 贴脸恐慌死**：敌已瞄我时下毒，当帧/下帧仍被击杀。→ p2: 敌瞄我时物理闪避，不下毒。(S5)
- **L-C poison 无跟进**：下毒后没补枪(脱离/敌离线)。→ p2: poison 只在已正对时放，下帧直接开火。
- **L-D 劣势星数换命对射(p2头号败因)**：敌先吃到唯一星(敌1我0)，随后双方同帧对射同归 → 星数判负我输。旧 insuranceLess 只在"图上还有星"时才拒绝换命，敌吃完星后 starExists=false 就去送命。→ p3: 改为比较 `myStars<=enemyStars` 才拒绝换命(干净枪照打，只闪避会同归的劣势对射)。**关键认知：干净击杀(敌没瞄我)无论星数都赢；只有"同归"才按星数判。**

## CHANGELOG
- **p1 (v2)**：v17 核心 + poison 锁定(≤5同线即下毒)。定级 17-3。
- **p2 (v3)**：
  1. poison 只在 `已正对敌 && 敌未瞄我 && dist≤5` 时放（进攻/有主动权）；不再对"瞄我的敌人"下毒。
  2. 贴脸威胁 escapeMove 改为**先物理闪避**，poison 仅在无路可退时兜底。
  目的：消除"下毒当帧被秒"和"下毒不跟进"的送头败因。
- **p3 (v4)**：星数感知换命(S6正确实现)。`losingMutual = 敌瞄我 && myStars<=enemyStars` → 拒绝对射、改闪避脱离去翻盘星数；干净枪(敌没瞄我)照常开火(无论星数都赢)。拦截射击层同步加此判。目的：消除 L-D。**实测消除了星数对射败(silver→gold3)。**
- **p4 (v5)**：BFS 路径缓存(globalThis)，只在星变/脱轨时重算，失败结果也缓存并退避8帧；动敌用greedy不用BFS。目的：消常规长局累计 runTime。**注意：map-55(Teleport Puzzle)等传送谜题图 runTime 是结构性的(极简无BFS bot也runTime)——无传送技能的坦克在那类图必被困，约占1%对局，接受放弃。**
